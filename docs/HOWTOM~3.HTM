This must surely be a FAQ but I don't find an answer in the FAQ list
for comp.lang.postscript. Is there way to merge several postscript
file into one .ps file? I'm not looking for anything fancy here (ie no
page order switching or other manipulations). Let's say I have files
f1.ps, f2.ps .... fk.ps; I'd like to produce a file all.ps which when
printed will give all the pages in the k files in the right order.

I realize the answer to this question may be very dependent on how the
files are generated. So here's more info. Each of the k files above is
produced as follows. Example:
   unix% gnuplot f1.gp; pstops 4:1@.5\(0in,5.5in\)+3@.5\(4.25in,5.5in\)+0@.5\(0in,0.0in\)+2@.5\(4.25in,0.0in\) foo.ps f1.ps
   # foo.ps is the file produced by `gnuplot f1.gp`
So f1.ps is the postscript output from a gnuplot file, altered so that
four pages are printed on one physical page. All k files are produced
this way, and so all have the same preamble. Shouldn't psmerge.pl in
the psutils suite work on this problem? I thought it would, but it
didn't; all psmerge.pl produced was a rather short file which appeared
to be the just the preamble of some file (though not the preamble common
to all the k input files).

I asked Jon Monsarrat (one time maintainer of the
comp.sources.postscript FAQ) about this, he suggested that simply
appending the files should work.
 unix% cat f1.ps .... fk.ps >> all.ps
This works (though you know this only after you print the file all.ps,
because `ghostview all.ps` shows only f1.ps), but only for a small
number of files --- in my case the limit is 23. After printing the
contents of f23.ps, the print job aborts with some stack error. I'm a
postscript novice so I can't say more about what that error is.


Suggestions greatly appreciated. Thanks,
-Anand.
-------------------------------------------------------

For completeness, I'm including file f1.ps, in case you need to know
more than what I've already said. 

%!PS-Adobe-2.0
%%Creator: gnuplot
%%DocumentFonts: Helvetica
%%BoundingBox: 50 50 554 770
%%Pages: 1 0
%%EndComments
%%BeginProcSet: PStoPS 1 13
userdict begin
[/showpage/erasepage/copypage]{dup where{pop dup load
 type/operatortype eq{1 array cvx dup 0 3 index cvx put

I asked this question a couple of days ago
and the answer I got from Michael Schindler (Thanks!)
was that you write the word "save" at the beginning
of the first file and the word "restore" at the end
of the first file. Then you just cat the documents
together.

I don't know if this want you want, but I worked great
for me!

    If you're printing on an Adobe Level 2 printer
    you can separate each job with,
    "true 0 startjob pop"

    This should almost always work,
    even if dictionaries have been loaded on the stack.
    It effectively restarts the PostScript interpreter between jobs.

    This is more reliable than surrounding each job with
    save and restore.

that will take a list of PS files and cat them together, by:

 2) stripping out the pages
 3) outputting the prolog with the %%Pages: corrected
 4) outputting each of the pages with the %%Page: corrected
