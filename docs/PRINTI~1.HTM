Printing Features in the User Interface

In Mac OS X, printing features can appear in the Print dialog user interface in three types of panes. From a user's perspective, the panes should look integrated, as the panes are distinguished by how the user interface is generated, not how the panes appear visually.

The type and number of panes vary depending on the printer's specific features and whether it is a PostScript printer. These are the three types of panes:

  • Standard panes. You get built-in panes for standard printing features "free" on Mac OS X. These are described in "Standard Panes".
  • Printer Features pane. This pane is available only for PostScript printers. It supports PostScript features that are not supported in a standard or custom pane. See "Printer Features Pane" for more information.
  • Custom panes. You can create a custom pane for a printer by writing a printing dialog extension (PDE). When you write a PDE for a PostScript printer, you need to specify, in the PDE's information property list, which PPD keywords specify the features that are implemented in the PDE. See "Adding PPD Keywords to a PDE's Information Property List".

    See Inside Mac OS X: Extending Printing Dialogs in Mac OS X for information on writing PDEs for PostScript and nonPostScript printers.


Standard Panes

Mac OS X supplies a number of panes in the Print dialog to support most standard printer features. Some of the features in the standard panes are available only for PostScript printers. PostScript-specific features are noted in the description of the panes in the following sections. The printing system uses information from the PPD file to populate the PostScript-specific features in the panes.


Copies & Pages

The Copies & Pages pane lets the user set the number of copies, specify whether copies should be collated, and set the range of pages to be printed. This pane is available to any printer, but collation is implemented differently for some PostScript printers. If the *Collate keyword is in the PPD file and is not constrained by the *UIConstraints keyword, collation is done by the PostScript printer. If the *Collate keyword is not in the PPD file, the printing system generates the print job so the output is pre-collated.


Figure 2-1 The Copies & Pages pane in the Print dialog


Layout

The Layout pane provides settings for the number of pages per sheet and the type of border on a page. For multiple pages per sheet, the user can also specify the layout direction—left to right then top to bottom, top to bottom then left to right, and so forth. This pane is available for all printers.


Figure 2-2 The Layout pane in the Print Dialog


Output Options

The Output Options is available for any printer, and provides the option to save a file as a PDF file or a PostScript file instead of sending it to a printer. The PostScript option is available only in Mac OS X version 10.1 and later, and only for PostScript printers.


Figure 2-3 The Output Options pane in the Print dialog


Paper Feed

The Paper Feed pane is available only for PostScript printers. There are two options in the Paper Feed pane: use paper from a single tray or use one tray for the first sheet and another tray for the remaining sheets. For either option, the user can select the trays. The tray option pop-up menus are populated by the values of the *InputSlot and *ManualFeed keywords in the PPD file for the printer. If the user chooses Auto Select, PostScript code associated with the *PageSize keyword is used to select the tray.


Figure 2-4 The Paper Feed pane in the Print dialog


Error Handling

The Error Handling pane is available only for PostScript printers. The Error Handling pane lets the user specify the action to be taken when certain errors occur. One setting determines how PostScript errors are handled and the other specifies to which tray to switch when the selected paper tray is empty. Tray switching options are enabled only when the PPD file contains the *TraySwitch keyword.


Figure 2-5 The Error Handling pane in the Print dialog


Duplex

The Duplex pane is available only for printers that provide the option to print on both sides of a sheet. Double-sided printing is specified by the *Duplex keyword in the PPD file for the printer.


Figure 2-6 The Duplex pane in the Print dialog


Summary

The Summary pane in the Print dialog displays a list of the currently selected printing options along with information that could be useful for troubleshooting (such as the printer's model name). The model name and PPD file version are displayed for PostScript printers whose PPD file contains values for the *ModelName and *FileVersion keywords. Display of the file version keyword is available only with Mac OS X version 10.1 and later.


Figure 2-7 The Summary pane in the Print dialog


Printer Features Pane

Printer Features is a pane in the Print dialog that is available only for PostScript printers. The pane contains any user interface feature specified in a PPD file that isn't one of the features in the Standard panes described in the previous section or in a custom pane, as long as the feature is defined as follows.

  • The feature must be defined with the *OpenUI/*CloseUI keywords.
  • The *OpenUI keyword must have either the Boolean or PickOne option. The PickMany option is not supported.
  • A Boolean option must explicitly define two options, True and False, otherwise it is treated as a PickOne option.
  • Features can be grouped by enclosing their definitions with the *OpenGroup/*CloseGroup keywords, and specifying a name for the group. This is optional.

The Mac OS X printing system follows these rules to construct the Printer Features pane:

  • Features are organized within the Printer Features pane under tabs, as shown in Figure 2-8. Tabs are labeled sequentially as Features 1, Features 2, and so forth, unless features are defined in groups. Then, the tabs are labelled with the group name. Figure 2-9 shows an example of tabs that are labelled with the group name specified using the *OpenGroup/*CloseGroup keywords.
  • The interface elements (checkboxes and pop-up menus) are displayed left-justified in the tabbed pane in the order in which they are defined in the PPD file.
  • No more than five features appear in a tabbed pane. If features are grouped and a group contains more than five features, new tabbed panes are created as needed. The additional tabs are labelled sequentially as Group Name 2, Group Name 3, and so forth.
  • A PickOne list is displayed in the interface as a pop-up menu.
  • A Boolean item is displayed in the interface as a checkbox. A checked item is True; an unchecked item is False.
  • A Boolean item that does not have two, and only two, options one of which is True and the other False, is displayed as a pop-up menu.
  • If a PickMany list is defined, it is ignored. This option is not currently supported.

Figure 2-8 A Printer Features pane in which the printing system groups the features

See "Specifying Grouping in the Printer Features Pane" for more information on how printer vendors can use grouping keywords to control the layout of the Printer Features pane.

If you plan to write a printing dialog extension (PDE) to create a custom interface, see "Adding PPD Keywords to a PDE's Information Property List". You need to register the PPD keywords that are supported by your custom printing dialog extension, otherwise the features associated with the keywords appear in the Printer Features pane.


Figure 2-9 A Printer Features pane in which groups are specified in the PPD file
