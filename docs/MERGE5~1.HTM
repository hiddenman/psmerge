: In general the answer is no.  If the PS files were created by different
: programs, or even different versions of the same program, the prolog will
: not be the same, therefore the files cannot be "put together".
Well, only if you try, you can indeed put things together.  I don't think that
there's any "algorithm" efficient enough to get the job done, but, as most of
you PS-hackers might know, the main difficulties are:
1) the translation and scaling differ in different files;
2) the transform matrix differ in different files.
To be simple, these problem can be overcome by pushing the transform matrix
of a main PSfile on stack, copy the initial matrix to the current matrix,
copy the sub-PSfile into the main file, and restore the transform matrix after
the sub-PSfile....
such process can be repeated as many times as you want.

However, it might be almost impossible if the two files you want to merge are
both multi-page PSfiles.  You will soon get lost of those matrix controlling
the individual files.

... and my humble opinion is that such a task would be quite useful.

... and my 2nd humble opinion is that in general the answer is yes. (i
    think). Here is what I tried, and it seems to work:

    1. assume both files are only 1 page long. 
    2. merge the two files.
    3. change the first 'showpage' (in what ever manner it appears, be
       it directly or via a defined procedure) to 'initgraphics'.
    4. try viewing/printing the file.

On a Unix platform, I think a shell script like the following should 
do the job in many cases:

file mergeps:

#!/bin/csh -f

sed 's/showpage/initgraphics/g' $1
cat $2

example usage:

mergeps file1.ps file2.ps > f1_f2.ps

or

mergeps file1.ps file2.ps | lpr -Pps

Whatever, you should get the idea.

I think this is another case of "in principle, you can't, in practice,
you can."  There are all sorts of things that can go wrong.  For
example, what if one file renames the "showpage" operator?  The
problem is that, in general, evaluting "3." above requies interpreting
every line of PS code.

OTOH, most well-behaved files do not redefine "showpage," or play
other nasty tricks.  So they can probably me merged by the system
above.
